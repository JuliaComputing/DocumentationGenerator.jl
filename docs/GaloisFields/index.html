Introduction

This module defines types representing [finite fields][galois-fields-wiki]. It supports both fields of prime order and of prime power order.

[galois-fields-wiki]: https://en.wikipedia.org/wiki/Finite_field

Synopsis

The easiest way to create Galois fields is with the @GaloisField and @GaloisField! macros. Typically, you use the former for a field of prime order and the latter for a field of prime power order. In the prime power case, you pass a display name / variable name for the primitive element.

using GaloisFields

F = @GaloisField 29     # ‚Ñ§/29‚Ñ§
G = @GaloisField! 27 Œ≤   # degree-3 extension of ‚Ñ§/3‚Ñ§; multiplicatively generated by Œ≤

F(2)^29 == F(2)
Œ≤^27 == Œ≤

The macros also accept pretty-printed versions: more difficult to type but more elegant to read:

F = @GaloisField ‚Ñ§/29‚Ñ§
G = @GaloisField ùîΩ‚ÇÇ‚Çá Œ≤

If you want to pass your own generator for the representation of a field of order $q = p^n$, you can:

F = @GaloisField! ùîΩ‚ÇÉ Œ≤^2 + Œ≤ + 2
Œ≤^2 + Œ≤ + 2 == 0

Lastly, there's also function interfaces in cases where macros are not appropriate:

F = GaloisField(29)               # ‚Ñ§/29‚Ñ§
G, Œ≤ = GaloisField(81, :Œ≤)        # degree-4 extension of ‚Ñ§/3‚Ñ§
G, Œ≤ = GaloisField(3, 4, :Œ≤)      # same; avoid having to factorize 81
F, Œ≤ = GaloisField(3, :Œ≤ => [2, 0, 0, 2, 1]) # same; pass our own custom minimum polynomial

Conversions

If you specify your own minimum polynomial, we make no assumptions about conversions between fields. For example, when defining

F = @GaloisField! ùîΩ‚ÇÇ Œ≤^2 + Œ≤ + 1
G = @GaloisField! ùîΩ‚ÇÇ Œ≥^2 + Œ≥ + 1

an operation like

G(Œ≤)

will throw an error. The mathematical reason is that the fields $F$ and $G$ are isomorphic, but there is two different isomorphisms. ("They are not canonically isomorphic.") To choose an identification, you can use the @identify macro (which is not exported by default, so we use its full path):

@GaloisFields.identify Œ≤ => Œ≥^2
@GaloisFields.identify Œ≥ => Œ≤^2

This allows for conversions such as

G(Œ≤)
convert(F, Œ≥ + 1)

The inner workings of this distinction are based on the symbol names. So if you define $F$ and $G$ with the same symbol and minimum polynomial:

F = @GaloisField! ùîΩ‚ÇÇ Œ≤^2 + Œ≤ + 1
G = @GaloisField! ùîΩ‚ÇÇ Œ≤^2 + Œ≤ + 1

then they are just considered equal and conversions work without extra work.

Conversions for the default minimum polynomials

If you do not specify a minimum polynomial, for example by using

F = @GaloisField! ùîΩ‚Çà‚ÇÅ Œ≤
G = @GaloisField! ùîΩ‚Çâ Œ≥

then we use [Conway polynomials][conway]. They have special compatibility relations between them, allowing conversions:

Œ≤^10 == Œ≥

This only works if there is a subfield isomorphic to $G$ in $F$, which is the case exactly when the order of $F$ is a power of the order of $G$.

[conway]: https://en.wikipedia.org/wiki/Conwaypolynomial(finite_fields)
