    Return a type representing a finite field.

    The single-argument signature returns the finite field $‚Ñ§/p‚Ñ§$.

    The two-arguments signature returns an algebraic extension of that field, with minimum polynomial given by the second argument: a dense representation of the univariate, monic polynomial, with ascending degree.

    The three-arguments signature returns an algebraic extension of that field, with minimum polynomial equal to the Conway polynomial for $(p,n)$. The GaloisFields package ships with a database of Conway polynomials and will raise an error if it does not contain an entry for $(p,n)$.

    Note that in the latter two cases, the variable name (e.g. Œ≤ above) is part of the type. This lets you define identifications between isomorphic (sub)fields. For example, with the following definition

    F = @GaloisField! ùîΩ‚ÇÇ Œ≤^2 + Œ≤ + 1
G = @GaloisField! ùîΩ‚ÇÇ Œ≥^2 + Œ≥ + 1

    the fields $F$ and $G$ are isomorphic, but not canonically. We might define

    @GaloisFields.identify Œ≤ => Œ≥ + 1
@GaloisFields.identify Œ≥ => Œ≤ + 1

    to allow for conversions like

    G(Œ≤)
convert(F, Œ≥ + 1)

    In the Conway case, you do not have to define your own identifications, as the Conway polynomials satisfy compatibility relations that allow us to use certain distinguished inclusions between them.
